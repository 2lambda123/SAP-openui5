Manual Integration Test: Flexible Column Layout - Replace-with Scenario

DESCRIPTION

This demo application is used to demonstrate how to synchronize data between a list report (left column) and an object page (middle column) by using the ODataModel#getKeepAliveContext API method. It also demonstrates how to replace an active version with a draft and vice versa.

PREPARATION

To check console errors open the developer tools before starting the application and check at each step that there are no errors and warnings.

Testing with mock data

Start the Draft application URL (adjust host and port if necessary):
  • local: http://<host>:<port>/test-resources/sap/ui/core/demokit/sample/common/index.html?component=odata.v4.Draft

REALIZE and CHECK

Edit & Cancel

  1. Select the first product in the list report.
  2. Check: An object page for the product appears, no input field is editable, only the Edit and Refresh buttons are enabled.
  3. Press the Edit button.
  4. Check: Some input fields are now editable, all buttons but Edit are enabled, and the list report shows that the first product is not an active entity.
  5. Enter 'Test' into the input field Name of the object page.
  6. Check: The list report shows the new name of the first product.
  7. Press the Cancel button.
  8. Check: No input field is editable, only the Edit and Refresh buttons are enabled, and the list report shows that the first product is an active entity with its original name.

Edit & Activate

  1. Press the Edit button.
  2. Check: Some input fields are now editable, all buttons but Edit are enabled, and the list report shows that the first product is not an active entity.
  3. Enter 'Test' into the input field Name of the object page.
  4. Check: The list report shows the new name of the first product.
  5. Enter a new quantity like '123' into the Quantity input field of the first part in the items table of the object page.
  6. Press the Save button.
  7. Check: No input field is editable, only the Edit and Refresh buttons are enabled, the list report shows that the first product is an active entity with a new name, and the items table of the object page shows the new quantity of the first part.

Cancel Without Edit

  1. Select the second product in the list report.
  2. Check: An object page for the product appears, some input fields are editable, all buttons but Edit are enabled, and the list report shows that the second product is not an active entity.
  3. Enter 'Test' into the input field Name of the object page.
  4. Check: The list report shows the new name of the second product.
  5. Press the Cancel button.
  6. Check: No input field is editable, only the Edit and Refresh buttons are enabled, and the list report shows that the second product is an active entity with its original name.
